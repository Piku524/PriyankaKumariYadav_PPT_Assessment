Task5: Deploying a Static Website in S3

Simple Definition
Deploy a static website by uploading files to an S3 bucket and making them publicly accessible.

Objective
To understand static website hosting and public access configuration in S3.

Platform
Amazon Web Services (AWS S3)

What I Did

Created an S3 bucket

Uploaded static website files (HTML, CSS, JS, images)

Enabled static website hosting

Disabled block public access

Added bucket policy for public access

Accessed website using S3 endpoint URL

Files Uploaded

index.html

style.css

script.js

images.jpg

Result

The static website is hosted successfully using S3 and can be accessed through a public URL.
